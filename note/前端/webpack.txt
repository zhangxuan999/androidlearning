webpack 处理各种依赖
webpack.config.json指定入口，出口。入口是main.js，出口是dist/bundle.js
package.json指定依赖的webpack版本，脚本
npm run build 跑build脚本，这个脚本就是执行webpack命令

css文件加载
1.在main.js中引用，commonJs引用方式。 require('./css/normal.css')
2.安装合适的loader，安装css-loader
3.在webpack.config.json配置modules对象下的rules，指定css文件用什么loader
4.同时还需要安装style-loader，程序运行才能起对应的效果

less文件加载
同上，第二步要安装less-loader 以及less

图片资源
安装对应的loader url-loader  npm install --save-dev url-loader
.在webpack.config.json配置modules对象下的rules，指定图片文件用什么url-loader，并且可以限制图片大小
加载图片小于limit，会将图片编译成base64字符串
加载图片大于limit，会编译报错，不能找到file-loader，需要安装file-loader，用file-loader来加载图片，且这张图片会打包到dist文件夹（发布）
但是dist目录下的这张图会重命名，用hash算法生成。
webpack.config.json配置publicPath 为"dist/",为图片url配置一个前缀路径


webpack.config.json 在 url-loader的option里面增加name属性 name：'img/[name].[hash:9].[ext]
img/name.hash:8.extension
